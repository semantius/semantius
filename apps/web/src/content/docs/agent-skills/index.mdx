**Agent Skills** let coding agents do real work on your Semantius platform: generate models, evolve schemas, configure RBAC, and deploy changes. They install into any agent that supports the Agent Skills format.

## What Skills Do

Each skill is a structured, repeatable procedure an agent can invoke:

- **[Business Analyst](/docs/models/create)**: build a custom model from scratch by interviewing the domain.
- **Schema Editor**: add, rename, or refactor entities and fields without breaking existing data ([Customize or Extend](/docs/models/create#customize-or-extend-an-existing-model) flow).
- **RBAC Configurator**: define roles, permissions, and access policies.
- **[Deployer](/docs/models/deploy)**: roll changes forward to a live environment when you're ready.
- **[Optimizer](/docs/models/optimize)**: reverse-engineer a live module back into a markdown spec to audit and refine.

Skills give agents a vocabulary and a workflow, instead of leaving them to improvise. The artifact they produce is a [semantic model](/docs/models) file.

## Installation

[Install the Semantius CLI and skills](/docs/agent-skills/installation) into your agent of choice. The skills are agent-agnostic: anything that supports Agent Skills can use them.

Among the agents we test against:

- **OpenClaw**
- **Claude Code**

If your agent supports Agent Skills, it works.

The skills also call into your platform's [MCP connectors](/docs/mcp-connectors) under the hood, so make sure those are wired up too.

## Ready-to-Use Models

Skills are also how ready-to-use models are installed and customized. You can pull a [published template](/docs/models/templates) into your platform, fork it, and let the skills handle the migration steps so the underlying data stays consistent.
